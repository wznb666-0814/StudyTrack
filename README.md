# 学迹 (StudyTrack)

学迹是一款面向学生的 Android 成绩管理与分析应用，强调沉浸式视觉与高效的数据记录体验。项目以 Kotlin + Jetpack Compose 构建，采用 MVVM 架构与本地离线存储，支持壁纸、主题色与液态玻璃风格的深度个性化。

## 功能概览
### 成绩与科目管理
- **科目管理**：为每个学生创建/维护学科，配置不同的满分（支持 100/70/60 以及其它分制），用于后续成绩录入与等级评定。
- **单次成绩记录**：为指定科目录入单次考试的分数、考试名称、考试类型（期中/期末/月考等）、日期，以及班排/年排/区排和反思内容。
- **批量成绩录入**：通过“批量添加成绩”页面，先填写一次考试的主题、日期与类型，然后为多个科目（如语文+数学+英语）分别录入本次考试的分数与排名，极大减少重复操作。
- **多用户模式**：内置轻量级用户系统，可为家庭中多名学生或同一设备的多名使用者建立独立的成绩与科目空间，通过设置中的“多用户模式”开启、切换与管理用户。

### 数据分析与可视化
- **趋势分析**：
  - 按科目聚合所有考试记录，展示该科目的分数变化趋势折线图。
  - 支持查看班级/年级/区排名的变化趋势（可选录入时填写）。
  - 每个科目卡片展示近期的平均分、最高分、最低分以及最新一次考试的等级。
- **成绩分布概览（首页）**：
  - 首页使用饼图统计最近一段时间内所有成绩在 A/B/C/D 等级上的分布比例。
  - 等级划分规则可由用户自定义（见“自定义等级算法”），分布图会实时跟随配置变化。

### AI 学习助手（DeepSeek 接入）
- **科目 AI 学习建议**：
  - 在“成绩分析”页面中，每一科目的成绩与排名趋势下方，都有“AI 学习建议”区域。
  - App 会基于该科目历史成绩曲线与班/年/区排名趋势，调用 DeepSeek 大模型自动生成 150–300 字左右的个性化学习建议。
- **AI 对话助手**：
  - 在每个科目卡片底部可以点击“对话”，进入专门的 AI 对话页面。
  - 模型会带着该科目的历史成绩与等级信息，与用户进行多轮对话，支持询问学习方法、复习规划、错题整理策略等。
  - 对话记录按照科目自动持久化存储，即使关闭应用或切换用户，重新进入该科目的对话页仍可看到历史聊天。
- **Markdown 展示优化**：
  - DeepSeek 输出的 Markdown 内容（如列表、加粗等）会在应用内以排版后的样式展示，而不会直接看到原始 `*`/`**` 符号。

### 自定义成绩等级算法
- **按满分分制配置等级阈值**：
  - 在“设置 → 自定义成绩等级算法”中，用户可以分别为满分 100/70/60 的科目配置 A/B/C 等级对应的起始分值（单位：分）。
  - 例如：
    - 100 分科目：A≥90，B≥80，C≥60；
    - 70 分科目：A≥60，B≥50，C≥42；
    - 60 分科目：A≥54，B≥45，C≥36。
- **全局生效**：
  - 分析页每个科目的“最新评级”会基于该科目的满分和当前配置的等级阈值实时计算。
  - 首页“成绩分布”饼图也会按新的 A/B/C/D 规则统计，方便直观查看当前成绩结构。

### 导入导出与备份
- **Excel 导出**：
  - 支持将当前用户的所有科目与成绩记录导出为 Excel（.xlsx）文件，用于备份或在其它工具中进一步分析。
  - 支持通过系统“共享”选择导出位置（本地存储 / 网盘等）。
- **Excel 导入**：
  - 支持从特定格式的 Excel 中批量导入成绩数据：
    - 首行标题要求固定格式（科目、考试名称、时间、分数、满分等）。
    - 导入时会按“科目名称”匹配已有科目，不存在时自动创建，并使用表格中的“满分”作为该科目的满分配置。
  - 导入过程中的错误会以对话框形式展示，并附带正确格式示例说明。

### 个性化与液态玻璃
- **壁纸与主题**：
  - 可选择自定义壁纸作为应用背景，支持按壁纸自动取主色，驱动整体配色。
  - 前景文字/图标支持“自动/强制亮色/强制暗色”三种模式，提升在不同壁纸下的可读性。
- **液态玻璃效果**：
  - 通过 Android Liquid Glass 库实现全局玻璃拟态卡片，支持实时调整：
    - 模糊强度、折射高度、折射幅度；
    - 玻璃染色强度、边框强度；
    - Vibrancy 效果与色差折射开关。
  - 在部分模拟器上会自动关闭高强度特效以避免渲染问题，真机则完整开启。

### 多用户与数据隔离
- **多用户模式**：
  - 在设置中开启后，可创建多个用户（如“孩子 A / 孩子 B / 自己”等），每个用户拥有独立的科目、成绩与 AI 对话记录。
  - 支持用户重命名、删除（至少保留一个用户）以及当前用户切换。
- **当前用户绑定**：
  - 科目与成绩记录在数据库层面都带有 `userId` 字段，所有查询和写入操作都会自动加上当前用户过滤，确保不同用户的数据不会混淆。

### 其他特性
- **离线优先**：所有数据（成绩、科目、设置、多用户信息、AI 对话历史）都存储在本地 Room / DataStore 中，除调用 DeepSeek API 时外，完全可以离线使用。
- **沉浸式 UI**：支持全屏壁纸背景、透明状态栏与导航栏、底部导航与卡片全部采用自定义玻璃风格。
- **批量操作友好**：在成绩列表中支持按科目与搜索过滤，并快速跳转到编辑 / 删除入口；新建成绩时可从最近记录中自动填充考试类型等信息（若已存在）。

## 技术栈
- 语言：Kotlin
- UI：Jetpack Compose (Material 3)
- 架构：MVVM
- 依赖注入：Hilt
- 数据库：Room (SQLite)
- 本地配置：DataStore
- 图表：自定义 Canvas 图表
- Excel：Apache POI
- 网络：OkHttp（DeepSeek API 调用）

## DeepSeek AI 配置
- **密钥安全**：
  - 代码中不再内置真实的 DeepSeek API Key，`BuildConfig.DEEPSEEK_API_KEY` 默认值为空字符串。
  - 建议为本项目单独创建一枚 DeepSeek 密钥，并在 DeepSeek 控制台定期轮换。
- **本地开发配置方式（推荐）**：
  1. 在项目根目录的 `local.properties` 中追加一行（示例）：
     - `DEEPSEEK_API_KEY=sk-xxxxxxx`
  2. Gradle 会在构建时将该值注入到 `BuildConfig.DEEPSEEK_API_KEY` 中，仅存在于本地构建产物中，不会被提交到 Git。
- **环境变量兜底**：
  - 若 `BuildConfig.DEEPSEEK_API_KEY` 为空，则 `DeepSeekClient` 会尝试读取系统环境变量 `DEEPSEEK_API_KEY`。
  - 在真机/模拟器上运行正式构建时，通常以 `local.properties` 方式注入即可，不必额外配置环境变量。
- **调用失败时的提示**：
  - 若密钥未配置或无效，会在 UI 层看到“DeepSeek 调用失败 / 请求超时”等友好文案。
  - 捕获到的 HTTP 错误会尽量解析出 DeepSeek 返回的 message 以便排查问题。

## 运行环境
- 最低系统：Android 10 (API 29)
- 目标/编译版本：Android 16（compileSdk = 36，targetSdk = 36）

## 快速开始
### 导入与同步
1. 打开 Android Studio
2. 选择 Open，定位到项目根目录（包含 `gradlew` / `gradlew.bat`）
3. 等待 Gradle 同步完成

### 运行调试
1. 连接真机或启动模拟器
2. 点击 Run (▶️)

### 命令行构建
```bash
./gradlew :app:assembleDebug
```

### 打包 APK
```bash
./gradlew :app:assembleRelease
```
构建产物位置：
- Debug：`app/build/outputs/apk/debug/app-debug.apk`
- Release：`app/build/outputs/apk/release/app-release.apk`
